## 📋 Deployment Steps

### 1. Push to GitHub
```bash
git add .
git commit -m "Ready for Render deployment"
git push origin main
```

### 2. Deploy Backend on Render
1. Go to [render.com](https://render.com)
2. Click "New +" → "Web Service"
3. Connect GitHub repository
4. Select `backend` folder
5. Use existing `render.yaml` configuration
6. Set environment variables:
   - `MONGO_URI`: Your MongoDB connection string
   - `DATABASE_URL`: Will be provided by Render PostgreSQL
   - `REDIS_URL`: Will be provided by Render Redis
   - `FRONTEND_URL`: Your frontend URL (after deployment)

### 3. Deploy Frontend on Render
1. Click "New +" → "Static Site"
2. Connect same GitHub repository
3. Select `web` folder
4. Use existing `render.yaml` configuration

### 4. Update Frontend API URL
After backend deploys, update the URL in `web/src/contexts/AuthContext.jsx`:
```javascript
baseURL: process.env.NODE_ENV === 'production' 
  ? 'https://your-actual-backend-url.onrender.com/api' 
  : 'http://localhost:10000/api'
```

## 🔧 Environment Variables Needed

### Backend
- `MONGO_URI`: MongoDB connection string
- `DATABASE_URL`: PostgreSQL connection (provided by Render)
- `REDIS_URL`: Redis connection (provided by Render)
- `JWT_SECRET`: Auto-generated by Render
- `FRONTEND_URL`: Frontend URL (after deployment)
